# Feedback Tips

Use this as lightweight guidance when the user asks how to collect real feedback from resume delivery, JD applications, interviews, or personal website sharing.

Do not treat feedback as a formal iteration loop yet. Most feedback is incomplete, polite, or distorted by context. Help the user collect better signals before changing direction or rewriting materials.

## Principle

The goal is not to ask "what do you think of my resume?" The goal is to collect specific, behavioral signals.

Good feedback is:

- Specific.
- From someone close to the hiring context.
- Based on the actual artifact used.
- Connected to a decision, concern, or hesitation.
- Repeatable across more than one source.

Weak feedback is:

- Generic praise.
- Polite rejection.
- "Looks good" from friends.
- Vague advice without role context.
- One-off comments from people outside the target market.

## What To Ask For

### After Resume Delivery

Ask:

- Which part of my background was clear within the first 20 seconds?
- Which part felt weak, unclear, or unsupported?
- Did my target direction look focused or scattered?
- Was there any claim you would challenge in an interview?
- If you would not move me forward, what is the main reason?

Avoid:

- "Can you help me look at my resume?"
- "Do you think this is good?"

### After An Interview

If the conversation felt reasonably positive, try to add the interviewer or recruiter on WeChat/LinkedIn and ask one focused follow-up.

Example:

```text
今天感谢你的时间。我想持续优化自己的表达，如果方便的话，想请教一个具体问题：从这次沟通看，我的经历里最需要补强或讲清楚的是哪一点？
```

Other useful questions:

- Which project did you find most relevant to this role?
- Which answer felt insufficient?
- Was the concern more about能力、经验深度、行业匹配，还是职级匹配？
- If I continue投类似岗位，最应该补强哪类证据？

### After Rejection

Ask once, politely. Do not argue.

```text
感谢反馈。方便的话，想请教一个具体问题：这次没有继续推进，主要是岗位匹配、经验深度、业务背景，还是表达材料的问题？我会用来优化后续求职。
```

If no response, treat it as no signal.

### From Friends Or Peers

Give them a concrete task:

- Pretend you are a hiring manager for [role].
- Read for 30 seconds.
- Tell me what three keywords you remember.
- Tell me which claim you do not believe yet.
- Tell me whether I look like [target role].

## Signal Classification

When feedback is available, classify it lightly:

```markdown
| Feedback | Source | Specificity | Likely Meaning | Confidence |
|---|---|---|---|---|
```

Confidence:

- High: specific, from target-market reviewer, tied to decision.
- Medium: specific but from adjacent reviewer.
- Low: vague, polite, or from non-target reviewer.

## How To Use Feedback

- Do not change positioning based on one low-confidence comment.
- If 2-3 credible sources repeat the same concern, update the relevant career asset.
- If feedback says "not enough experience", identify whether the problem is actual gap, hidden evidence, or weak expression.
- If feedback says "not a fit", check whether the JD was misaligned with the user's target direction.
- If feedback is only generic praise, do not optimize based on it.
