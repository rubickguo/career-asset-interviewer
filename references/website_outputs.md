# Website Outputs

Use this when producing personal website or portfolio content.

## Site Structure

Recommended first version:

- Hero: name, positioning, 1-sentence value proposition.
- Proof strip: 3-5 keywords or outcomes.
- Featured projects: 2-4 project case studies.
- Skills with evidence: not a generic skill list.
- About: concise personal narrative.
- Contact / links.

## Project Case Study

```markdown
## Project Title

### Context

### My Role

### What I Did

### Result

### What It Shows
```

## Copy Rules

- Write for the user's target audience: recruiter, hiring manager, potential client, collaborator, or investor.
- Make the positioning obvious in the first screen.
- Use concrete project evidence rather than adjectives.
- Keep claims aligned with confirmed facts.

## Simple HTML Output

If the user asks for an actual site and no framework is specified, create a single static HTML file with embedded CSS. Keep it clean, responsive, and easy to edit.
