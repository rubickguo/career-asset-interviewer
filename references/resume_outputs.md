# Resume Outputs

Use this when generating resume material from the career asset library.

## Rules

- Start from confirmed positioning keywords.
- Use the user's strongest evidence first.
- Prefer concrete scope, action, result, and relevance.
- Do not fabricate metrics. If a metric is missing, ask for it or use truthful non-numeric impact.
- Keep bullets aligned with the target direction or JD.
- Mark unconfirmed details as "待确认" outside the final resume draft.

## Bullet Formula

Use one of these patterns:

```text
在 [业务/产品/团队背景] 下，负责 [范围]，通过 [关键动作/方法]，实现/推动 [结果]，体现 [关键词能力]。
```

```text
主导/参与 [项目]，解决 [问题]；完成 [动作]，带来 [业务/效率/质量/用户/团队影响]。
```

```text
围绕 [目标] 搭建/优化 [机制/流程/产品/系统]，提升 [指标/体验/协作效率]，支撑 [业务结果]。
```

## Resume Variants

Common variants:

- General resume: broad but keyword-clear.
- JD-tailored resume: mirrors the JD's highest-priority needs using truthful evidence.
- Transition resume: emphasizes transferable skills and proof.
- Management resume: highlights ownership, decision-making, leverage, cross-team impact.
- Specialist resume: highlights depth, craft, technical/process expertise.

## Output Sequence

1. State target positioning.
2. List chosen keywords.
3. List evidence used.
4. Draft the resume section or bullets.
5. List what still needs confirmation.

## Quality Check

Before finalizing, check:

- Does every important keyword have evidence?
- Are the strongest projects placed early?
- Is the wording too generic?
- Does any claim exceed the evidence?
- Would the target reader understand the user's value within 20 seconds?
