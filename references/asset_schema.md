# Asset Schema

Use these structures when creating or updating the user's career asset library.

## profile.md

```markdown
# Profile

## Current Goal

## Target Directions

## Wants

## Does Not Want

| Item | Reason | Type | Solvable? | Notes |
|---|---|---|---|---|

## Strengths

## Preferences

## Constraints

## Confirmed Facts

## Open Questions
```

## directions.md

```markdown
# Direction Ranking

## 1. Direction Name

- Type: high-certainty / growth / exploration / deprioritized
- Why it fits:
- Evidence:
- Gaps:
- Risks:
- Next experiment:
```

## keywords.md

```markdown
# Positioning Keywords

| Keyword | Priority | Evidence Strength | Supporting Projects | Missing Proof | Status |
|---|---|---|---|---|---|
```

Status values:

- confirmed
- likely
- weak
- 待确认
