# Metrics And Evidence

Use this when selecting or rewriting resume evidence.

## Principles

- Results should be placed before process when possible.
- Metric names must be meaningful to an external hiring reader.
- Avoid internal-only metrics unless the business meaning is explained.
- If a result metric is unavailable, use a process metric.
- If both result and process metrics are weak, reorder the resume so stronger quantified projects appear first.

## Metric Priority

1. Business result: revenue, conversion, retention, paid users, cost saved, quality improvement.
2. User result: satisfaction, NPS, completion rate, usage, content consumption, creator participation.
3. Operational result: time saved, error reduction, review cost, automation rate, throughput.
4. Process metric: content volume, number of assets processed, number of users/creators supported, cadence, coverage.
5. Activity count: meetings, documents, features shipped. Use only when tied to a result.

## Good Metric Questions

- 这个结果的上层目标是什么？
- 这个指标影响了什么业务结果？
- 如果最终结果不明显，过程指标有没有变化？
- 反作弊、审核、内容治理、工具提效分别有什么可验证变化？
- 优质内容曝光率、审核通过率、配置错误率、处理时长、满意度、付费率有没有前后对比？
- 这个数字面试官追问口径时，用户能否解释清楚？

## Wangkun Notes To Preserve

- "达成率" often has weak external meaning. Prefer growth rate or absolute business impact, such as `YoY +XX%`.
- Do not write only "what was done"; emphasize result and what was different about the user's action.
- Avoid falling into descriptions everyone can claim.
- Put result data in front, then explain process.
- If the outcome metric is poor or unavailable, describe process indicators and the operational value created.
- More metrics are useful only when they are meaningful and explain impact.

## Bullet Rewrite Pattern

Weak:

```text
负责审核流程优化，制定审核标准并推动落地。
```

Stronger:

```text
将攻略审核从人工经验判断改为“标准沉淀 -> AI 初筛 -> 人工复核”的流程，优质攻略占比由 X 提升至 Y，人工审核成本由 X 降至 Y。
```

If result is unavailable:

```text
建立“标准沉淀 -> AI 初筛 -> 人工复核”流程，将审核口径从个人经验转为可复用标准，覆盖 X 类内容和 Y 个审核场景。
```
