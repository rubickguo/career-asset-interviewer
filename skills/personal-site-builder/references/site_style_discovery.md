# Site Style Discovery

## Questions

- Which sites feel closest to your taste?
- Which sites feel impressive but wrong for you?
- Should the site feel recruiter-oriented, portfolio-oriented, creator-like, founder-like, or consultant-like?
- Do you prefer dense professional proof or expressive personal narrative?
- Should the site be quiet, editorial, technical, playful, premium, or experimental?
- Is the user presenting as a C-end product person, B-end product person, technical builder, operator/growth person, or manager? These should produce different websites.

## Product-Type Differences

### C-End Product

Usually needs:

- Stronger personality and taste signal.
- User empathy and product sensibility.
- Visual examples, prototypes, user stories, interaction details.
- More attention to emotional resonance and public-facing polish.

### B-End Product

Usually needs:

- Clear structure, credibility, and system thinking.
- Workflow diagrams, before/after process, stakeholder context.
- Dense but readable project evidence.
- Less decorative expression; more decision logic and measurable impact.

### Technical Builder

Usually needs:

- Shipped projects, architecture, tools, GitHub links, usage evidence.
- Clear distinction between demo and adopted tool.
- Build quality and maintainability signals.

### Manager / Lead

Usually needs:

- Scope, team leverage, decision-making, resource coordination.
- Fewer isolated feature details; more portfolio of outcomes.

## Resume Vs Website Boundary

- Resume should be concise, conservative, and low-risk.
- Website can show more personality, demos, product thinking, visual process, and public artifacts.
- Do not expose sensitive company information, unreleased materials, private data, or politically risky framing.
- If a project is useful but sensitive, show sanitized process and learning instead of raw materials.
