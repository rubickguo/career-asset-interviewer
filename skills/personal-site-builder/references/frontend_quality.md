# Frontend Quality Checklist

Use this when building or reviewing a personal website. If a dedicated `frontend-design` skill is available in the environment, use that skill as the primary design reviewer and use this file as a fallback.

## Layout

- First screen must communicate the user's positioning.
- Avoid generic template feeling.
- Avoid cards inside cards.
- Do not use large decorative sections that hide evidence.
- Make spacing, alignment, and hierarchy consistent.
- Ensure text does not overlap or wrap awkwardly.

## Audience Fit

- C-end product sites can show more taste, prototypes, and emotional product sense.
- B-end product sites should prioritize structure, system thinking, workflows, and measurable outcomes.
- Technical builder sites should surface shipped projects, code, architecture, and real usage.
- Management-oriented sites should show scope, leverage, tradeoffs, and organizational outcomes.

## Visual Quality

- Use meaningful visual assets when available.
- Do not rely on generic gradients or decorative blobs.
- Typography should be readable and professional.
- Color choices should support the user's target identity.
- Mobile layout must be checked.

## Evidence

- Each project should show context, user role, key action, result, and what it proves.
- Do not use vague adjectives without proof.
- Links and project artifacts should be easy to find.
