# 优化清单

这是后续逐项优化的工作清单。当前先完成 skill 拆分，不在本轮解决所有细节。

## P0：深访问题树（已完成第一版）

- 把“问题库”升级成“问题树”。
- 按行业、职能、职级分支。
- 每个问题要有判断目的、可能回答、追问路径和产出字段。
- 避免让 LLM 自由发挥导致访谈质量波动。
- 已落地到 `skills/career-direction-interviewer/references/question_tree.md`。
- 根据产品讨论调整：加入“已有简历”入口；默认优先判断是否延续当前职业；第一版只覆盖互联网产品、运营、程序员，并辅助考虑管理类岗位；AI/游戏作为领域叠加。

## P0：JD 可选附加流程（已完成第一版，已按产品逻辑修正）

- 核心流程仍然是：先职业方向深访，再优化简历。
- JD 分析是附加且可选流程，不是默认前置流程。
- 未完成深访时，即使用户提供 JD，也只能用来询问“是否真的想投这个岗位”，不能直接做完整匹配。
- JD skill 只负责 JD 拆解、定性匹配、客观劝退/建议和简历调整策略，不负责面试故事。
- 已落地到 `skills/jd-fit-strategist/SKILL.md` 和 `references/jd_capability_model.md`。

## P0：反馈迭代闭环（暂缓，降级为反馈获取 Tips）

- 支持投递反馈、HR 反馈、面试失败原因、用户主观反馈回流。
- 建立“反馈 -> 诊断 -> 修改关键词/故事/简历”的路径。
- 这是职业辅导比一次性交付更有价值的部分。
- 当前判断：反馈来源和真实性难以保证，暂不做正式闭环。
- 已降级为 `shared/feedback_tips.md`，只提示用户如何获取更真实、更具体的反馈。
- 未来如果有足够真实样本，再升级为正式迭代流程。

## P1：输出范围收敛（已完成第一版）

- 核心优先做深：职业定位、简历素材、面试故事。
- 个人网站后置，不进入默认流程。
- 避免一个 skill 同时扮演职业规划师、简历顾问、面试教练、前端开发。
- 已落地到 `shared/output_boundaries.md`。
- 已确认：`career-direction-interviewer` 和 `resume-story-builder` 是主流程；`jd-fit-strategist` 和 `personal-site-builder` 是可选辅助流程。
- 所有 skill 都必须先读取用户画像/职业资产，再做建议或输出。
- 已补充：个人网站也必须读取用户画像，并根据 C 端产品、B 端产品、技术建设者、管理者等身份差异设计。
- 已补充：简历生成应走 HTML 预览 -> PDF 导出 -> PDF/渲染检查流程，避免大片空白、单字一行、不可读字体和错误分页。
- 已补充：简历深访应挖掘与目标岗位相关的生活经历，例如游戏岗位中的职业战队、高分段玩家、社区组织等。
- 已补充：指标优先结果指标，其次过程指标；指标名称必须对外部招聘方有意义。

## P1：行业/职能差异化

- 产品、运营、设计、研发、市场、销售、管理岗需要不同追问。
- 互联网、游戏、制造业、咨询、教育等行业叙事不同。
- 同一项目在不同目标方向下应提炼不同证据。

## P1：主叙事选择机制

- 简历预解析后生成 3-5 个职业主叙事候选。
- 问用户“哪个是你想成为的人，哪个只是你做过但不想被定义成的身份”。
- 主叙事决定后续项目深挖和简历排序。

## P1：证据强度评级

- 每个关键词都要有证据强度：strong / medium / weak / missing。
- 弱证据关键词不能直接进入主简历。
- 缺证据但有潜力的方向进入“小实验”。
- 已部分落地到 `shared/career_asset_schema.md`：关键词、项目、指标都加入证据强度/置信度字段。

## 已从 rubick-career-portfolio 记忆文件蒸馏

- 项目卡升级：增加简历优先级、作品集优先级、公开展示建议、敏感风险、结果指标、过程指标、证据材料、可迁移能力。
- 简历与网站边界：简历保守、低风险、筛选友好；网站可展示 demo、过程、审美和产品思考。

- 生活经历：与目标岗位相关时可作为证据，例如游戏岗位中的职业战队、高分段玩家、社区组织。
